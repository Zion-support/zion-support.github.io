Before enabling autonomy in customer-facing pathways, use this checklist:

1. Define precise permissions and tool scopes
2. Implement least-privilege credentials and time-bound tokens
3. Add eval-gated release checks with rollback hooks
4. Monitor with real-time anomaly signals and capture evidence
5. Provide human-in-the-loop escape hatches for edge cases

This document includes a downloadable template you can adapt for SOC2-friendly change reviews.
