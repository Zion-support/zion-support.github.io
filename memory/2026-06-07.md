## Deploy Fix — Critical Lesson
The site deploys via GitHub Actions workflow `gh-pages.yml` which:
1. Triggers on push to `main`
2. Runs `npm install && npm run build`
3. Uploads `out/` directory as Pages artifact
4. Deploys via `actions/deploy-pages@v4`

The `gh-pages` branch is NOT used for deployment — it's a legacy leftover.
The Pages API `gh api .../pages/builds --method POST` just triggers a redeploy of whatever was last built by the workflow.

**Never push `out/` to `main` again.** The workflow handles it.
