### 🎯 Next Proposal — #57: Autonomous Meta Tags Quality Scanner

**Problem:** Meta titles and descriptions are critical for SEO. Common issues:
- Titles too short/long (>50–60 chars)
- Duplicate descriptions across pages
- Missing canonical URLs
- Incorrect robots.txt directives (`noindex` on key pages)
- Missing Open Graph tags (complementary to #53)

No automated quality gate exists for these basic SEO signals.

**Solution:** Weekly meta tags scanner:
- Crawl all pages (via build output or local server)
- Extract `<title>`, `<meta name="description">`, `<link rel="canonical">`, `<meta name="robots">`
- Validate:
  - Title length: 50–60 chars ideal; warn >70; fail >80
  - Description: unique per page; length 120–160 chars; not empty
  - Canonical: present on all pages; points to correct URL
  - Robots: no `noindex` on homepage/category pages unless intentional
  - OG tags: `og:title`, `og:description`, `og:image` present (ties to #53)
- Open GitHub issues on violations with fix suggestions
- Telegram weekly digest with quality score (%)

**Components:**
- `automation/meta-tags-scanner.cjs` — Cheerio/Playwright page fetch; rule engine; issue creator
- `.github/workflows/meta-tags-scanner.yml` — Weekly Wednesday 11:00 UTC + manual
- `docs/META-TAGS-SCANNER.md`
- State: `.hermes/memory/meta-tags-scanner/` (baseline.json, issues.json)

**Config (env):**
```bash
META_TITLE_MIN=50
META_TITLE_MAX=60
META_DESC_MIN=120
META_DESC_MAX=160
META_CHECK_CANONICAL=true
META_CHECK_ROBOTS=true
META_OPEN_ISSUES=true
```

**Safety:** Read-only page fetch; no code mutation; opens issues; deterministic string length/uniq checks.

**Standing:** Ready for autonomous implementation.
