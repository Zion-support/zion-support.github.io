# Safe Change Management for AI Agents (2026)

This blueprint shows how to ship AI and agent changes safely using:

- Evaluation gates in CI/CD and runtime
- Progressive delivery (shadow, canary, cohort)
- Automated rollback and incident playbooks
- Evidence capture for audit (evals, incidents, approvals)

Key outcomes: higher reliability, faster recovery, and audit‑ready changes.

## Core Practices

1. Define policy‑as‑code for change types and risk tiers
2. Require passing evals for merge and release gates
3. Automate rollbacks with preapproved, tested procedures
4. Capture evidence in a production Evidence Hub

## Metrics

- Change failure rate < 3%
- Median rollback time < 5 minutes
- Gate pass rate > 95%
