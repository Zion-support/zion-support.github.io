Reliable autonomy requires repeatable operations. This guide provides ready‑to‑adopt runbook patterns and checklists tied to evaluation gates so teams can react quickly and prevent regressions.

Core runbooks:

- Incident triage with eval‑gated mitigations
- Canary promotion with scorecard approvals
- Rollback and route throttling on policy violation
- Post‑incident review with scenario additions

Operator checklists:

- Pre‑release gate score thresholds defined and reviewed
- Canary success criteria met (quality, latency, cost)
- Evidence Hub updated with tracing, evals, and approvals
- Regression scenarios added for caught issues

KPIs to monitor:

- MTTR and incident rate by severity
- Gate pass rate and change failure rate
- Cost per successful task and latency p95

Adopt these runbooks to make autonomy predictable, auditable, and business‑safe.
