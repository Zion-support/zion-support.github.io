# Quick Start Guide - Intelligent PM2 Automation System

## 🚀 Get Started in 5 Minutes

### 1. Start the Intelligent System

```bash
npm run pm2:intelligent:start
```

### 2. Check System Status

```bash
npm run pm2:intelligent:status
```

### 3. Monitor in Real-Time

```bash
npm run pm2:intelligent:monitor
```

## 📊 What's Running

The intelligent system includes:

- **AI Health Monitor** - Monitors system health every 30 seconds
- **Git Automation** - Automatically merges branches and resolves conflicts
- **Performance Optimizer** - Optimizes performance every hour
- **Security Scanner** - Scans for vulnerabilities every 4 hours
- **Code Quality Monitor** - Checks code quality every 2 hours
- **Deployment Automation** - Handles deployments with rollback
- **Resource Monitor** - Monitors system resources every 5 minutes
- **Error Recovery** - Automatically recovers from errors
- **Backup System** - Creates daily backups

## 🔧 Useful Commands

```bash
# View all logs
npm run pm2:intelligent:logs

# Restart the system
npm run pm2:intelligent:restart

# Stop the system
npm run pm2:intelligent:stop

# Health check
npm run pm2:intelligent:health
```

## 📁 Key Files

- `ecosystem.intelligent.cjs` - Main PM2 configuration
- `scripts/start-intelligent-pm2-system.sh` - Startup script
- `scripts/intelligent/` - All automation scripts
- `logs/` - System logs and reports

## 🚨 Alerts & Monitoring

The system will automatically:

- Alert you to high CPU/memory usage
- Fix common code issues
- Resolve merge conflicts
- Optimize performance
- Scan for security vulnerabilities
- Create backups
- Recover from errors

## 📊 Reports

Check these directories for detailed reports:

- `logs/` - Health and performance metrics
- `security-reports/` - Security scan results
- `quality-reports/` - Code quality reports
- `performance-reports/` - Performance analysis
- `backups/` - Automated backups

## 🔧 Configuration

Edit `.env` file to customize:

- Monitoring intervals
- Alert thresholds
- Auto-fix settings
- Backup retention

## 🆘 Troubleshooting

If something goes wrong:

1. Check logs: `npm run pm2:intelligent:logs`
2. Check status: `npm run pm2:intelligent:status`
3. Restart system: `npm run pm2:intelligent:restart`
4. Check health: `npm run pm2:intelligent:health`
